# 📄 論文翻譯 - Implicit Execution Tracing for Multi-Agent Attribution

> 原文：[arXiv:2603.17445](https://arxiv.org/abs/2603.17445)
> 翻譯：2026-03-19

---

## 摘要

**研究問題**：
當多 Agent 系統產生錯誤或有害答案時，如果沒有執行日誌和 Agent 標識，如何追究責任？

**解決方案**：
提出 **IET (Implicit Execution Tracing)**，一個元數據無關的框架：
- 從生成的文字直接進行 Token 級別歸因
- 簡單的互動拓撲重建機制

**核心創新**：
- 在生成過程中，將 Agent 特異的密鑰信號嵌入 Token 分佈
- 文字轉變為自我描述的執行追蹤（僅可用密鑰檢測）
- 轉換感知評分方法識別 Agent 交接點

---

## 核心發現

### 1. 問題背景

| 挑戰 | 說明 |
|------|------|
| 責任歸屬 | 多 Agent 協作誰負責？ |
| 日誌缺失 | 沒有執行記錄如何追蹤？ |
| 隱私審計 | 如何保護隱私同時審計？ |

### 2. 技術架構

```
生成時 (Generation)
┌─────────────────────────────┐
│  Agent-specific Keys        │  ← 每個 Agent 有獨立密鑰
│  → 嵌入 Token 分佈         │
└──────────────┬──────────────┘
               │
               ▼
    文字變為「自我描述的執行追蹤」

檢測時 (Detection)
┌─────────────────────────────┐
│  Transition-aware Scoring   │  ← 識別交接點
│  → 重建互動圖              │
└─────────────────────────────┘
```

### 3. 關鍵能力

1. **Token 級歸因**
   - 直接從最終文字識別每個 Agent 的貢獻

2. **互動圖重建**
   - 識別 Agent 交接點
   - 重建協作結構

3. **隱私保護審計**
   - 無需保留執行日誌
   - 僅需密鑰即可驗證

1. **執行追蹤增強**

2. **責任歸屬**

   - 自動生成審計報告

3. **隱私保護**
   - 最小化日誌存儲
   - 加密歸因數據
