## 摘要

**研究問題**：
科學 Python 程式碼中的方法論錯誤會產生看似合理但不正確的結果，傳統 linter 和靜態分析工具無法檢測到。

**解決方案**：
提出 **scicode-lint**，採用雙層架構：
- **建構時**：使用前沿模型生成模式
- **執行時**：使用小型本地模型

**成果**：
- 預處理洩漏檢測：100% 召回率下達 65% 精確率
- 38 篇科學論文測試：62% 精確率
- 66 種模式測試：97.7% 準確率

---

## 核心發現

### 1. 問題背景

| 問題類型 | 說明 | 檢測難度 |
|----------|------|----------|
| 資料洩漏 | 訓練/測試資料未分離 | 高 |
| 交叉驗證錯誤 | 驗證策略不當 | 高 |
| 隨機種子缺失 | 結果不可重現 | 中 |

### 2. 技術架構

```
建構時 (Build Time)
┌─────────────────────────┐
│   Frontier Model        │  ← GPT-4, Claude
│   (Pattern Generation)  │
└───────────┬─────────────┘
            │ Patterns
            ▼
執行時 (Runtime)
┌─────────────────────────┐
│   Small Local Model     │  ← Phi, TinyLlama
│   (Detection)          │
└─────────────────────────┘
```

### 3. 關鍵創新

1. **模式生成，非手動編碼**
   - 適應新函式庫版本只需 token，不用工程時間

2. **雙層架構**
   - 建構時複雜，執行時輕量

3. **LLM 判斷**
   - 使用 LLM 作為評估標準
