## Lesson Learned Playbook 完成 (15:19)

Johnny 完成了 Lesson Learned Playbook，包含：

### 6 大核心教訓

1. **開發流程**
   - 不要先開發後補文件 → 先制定 spec，再開發
   - 基準要一致 → 不要同時改多個變數
   - 變因要單一 → 一次只改一個變數

2. **Constitution 使用**
   - 有 ≠ 使用 → 要真的執行 CLI
   - 有文件 ≠ 有使用 → 要真的在開發中應用
   - 版本要確認 → 使用最新版本 v5.21+

3. **外部 spec**
   - 基準一致 → 所有版本套用相同 spec
   - 模組分離來自 spec

### 正確流程 (V12 實踐)
1. Constitution CLI → 設定閾值
2. 外部 spec → 制定規格
3. 開發 → 遵從 spec
4. Quality Gate → 把關
5. 交付 → 測試通過
