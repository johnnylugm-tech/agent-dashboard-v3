## ASPICE 4.0 學習摘要

### 背景
ASPICE 3.1 自2017年以來是汽車軟體開發的標準，但隨著：
- AI/ML 進入汽車應用
- SDV (Software Defined Vehicles) 興起
- 網路安全需求增加
- ISO 26262 / UN R155 法規要求

ASPICE 4.0 於 2023年12月正式發布，2025年3月31日後全面採用4.0標準。

### 主要變化

| 維度 | ASPICE 3.1 | ASPICE 4.0 |
|------|------------|------------|
| 範圍 | 軟體為主 | 整個機電系統（軟體+硬體+ML） |
| 硬體流程 | 無 | HWE.1 ~ HWE.4 (4個) |
| ML流程 | 無 | MLE.1 ~ MLE.4 (4個) |
| 安全/網安 | 一般指引 | 強化對齊 ISO 26262 / ISO 21434 |
| 敏捷支援 | 無 | 正式支援敏捷/DevOps |
| 評估標準 | ISO 15504 | ISO/IEC 33002:2015 |

### 新增流程區域

#### 硬體工程 (HWE)
- HWE.1 - 硬體需求分析
- HWE.2 - 硬體設計
- HWE.3 - 硬體實現
- HWE.4 - 硬體驗證

#### 機器學習工程 (MLE)
- MLE.1 - ML資料管理
- MLE.2 - ML模型開發
- MLE.3 - ML整合
- MLE.4 - ML驗證

### 遷移路線圖

| 階段 | 時間 | 內容 |
|------|------|------|
| Phase 1 | 1-2月 | 現狀分析 + Gap Analysis |
| Phase 2 | 1-2月 | 培訓 + 認知建立 |
| Phase 3 | 2-3月 | 流程更新 |
| Phase 4 | 3-4月 | 試點項目 |
| Phase 5 | 2-3月 | 全組織推廣 |

總工期：6-18個月

### 與 methodology-v2 的關聯

| ASPICE 4.0 概念 | methodology-v2 對應 |
|----------------|-----------------|
| 流程定義 | 標準化開發流程 |
| 驗證 (Verification) | AutoQualityGate |
| ML驗證 | Agent Evaluation |
| 追溯性 (Traceability) | 訊息追蹤 |
| 安全/網安 | Governance / Security |

### 參考資源
- https://autosar.io/en/insights/aspice-40-vs-31-migration
