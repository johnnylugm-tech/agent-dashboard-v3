## 成功關鍵因素

### 1. 做對的事情

| 因素 | 說明 | 範例 |
|------|------|------|
| **結構化思考** | 不是給答案，而是給方法論 | TDD, brainstorming, systematic-debugging |
| **強制流程** | 每次都執行相同步驟 | deploy-checklist, code-reviewer |
| **產出確定** | 結果可預期、可驗證 | test-harness, security-scan |
| **實用優先** | 解決實際問題 | docker-optimize, frontend-design |

### 2. 不做的事情

| 不做 | 原因 |
|------|------|
| **不給模糊建議** | 「可以考虑」→ 具體步驟 |
| **不依賴特定框架** | 通用方法論 |
| **不要求複雜安裝** | 一分鐘內可用 |
| **不只是提示** | 是完整工作流 |

### 3. 關鍵特點

| 特點 | 重要性 |
|------|---------|
| **Markdown 格式** | 易讀、易用 |
| **決策樹/檢查清單** | 結構化輸出 |
| **依賴少** | 零門檻 |
| **可測試** | 結果可驗證 |

---

## 成功公式

```
成功 = 結構化方法論 × 產出確定性 × 易用性
```

## 對我們專案的啟示

### 當前問題

| 問題 | 影響 |
|------|------|
| 太大太重 | 下載量低 |
| 依賴複雜 | 使用門檻高 |
| 產出不確定 | 吸引力低 |
| 缺乏結構 | 價值不明顯 |

### 需要調整

| 調整方向 | 具體行動 |
|----------|----------|
| **縮小範圍** | 只做一個核心功能 |
| **降低依賴** | 零依賴或最小依賴 |
| **產出確定** | 每個步驟有明確輸出 |
| **格式標準** | SKILL.md 格式 |
