# 台灣高鐵自動化訂票系統 - 技術規格書

## 1. 專案概述

### 目標
開發高鐵自動訂票系統，支援高併發環境，需符合台灣法規。

### 技術選型
- 協定模擬模式：Requests + asyncio
- 瀏覽器自動化：Playwright
- 非同步：aiohttp + 指數退避

---

## 2. 功能模組

### 2.1 查詢引擎 (Query Engine)
- 站代碼轉換 (台北 1000 ~ 左營 1070)
- 班次篩選：時段、車次、早鳥折扣
- TDX API 整合 (ptx.transportdata.tw)

### 2.2 驗證碼辨識 (CAPTCHA Solver)
- CNN OCR 模型
- 灰階、去噪預處理
- 歪斜、扭曲處理
- 自動重整直到成功

### 2.3 訂位執行 (Booking Executor)
- 參數驗證（身份證號格式）
- Session/Cookie/CSRF 管理
- User-Agent 偽裝

### 2.4 業務邏輯
- 預售窗計算（一般日 29 天 / 週末延長）
- 付款期限監控

---

## 3. 預售期規範

| 情境 | 預售期 | 備註 |
|------|--------|------|
| 一般平日 | 29天內（含乘車當日） | 起點 00:00 AM |
| 週五/週六 | 可預訂至四週後之週日 | 提前開啟 |
| 疏運期間 | 依官網公告 | 春節、連假 |
| 網路訂票截止 | 發車前 1 小時 | - |
| App 訂票截止 | 發車前 5 分鐘 | - |

| 條件 | 付款期限 |
|------|----------|
